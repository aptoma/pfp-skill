# HTML Output Reference

This document describes the complete HTML structure that Gazette produces. It is the primary reference for writing SCSS stylesheets.

## Document Structure

```html
<!DOCTYPE html>
<html lang="no">
  <head>
    <meta charset="UTF-8">
    <title>Gazette: {edition title}</title>
  </head>
  <body class="{body classes}" style="{CSS root properties}">
    <style>
      /* shared base styles (styles.css) */
      /* theme stylesheet */
      .frontpage {
        width: {trim.width}mm;
        height: {trim.height}mm;
      }
      @page {
        margin: 0; padding: 0;
        size: {trim.width}mm {trim.height}mm;
        bleed: {bleed}mm;
        marks: crop -ro-bleed;
      }
    </style>
    <div class="frontpage" style="--margin-top: ...; --margin-right: ...; --margin-bottom: ...; --margin-left: ...">
      <div class="content-area">
        <!-- blocks rendered here -->
      </div>
    </div>
  </body>
</html>
```

Source: `server/handlers.js:renderStatic()`

## Body Classes

Generated by `src/lib/utils.js:getBodyClasses()`:

| Class | Source | Example |
|-------|--------|---------|
| `product--{name}` | Product name | `product--acme-print` |
| `print-product--{name}` | Print product name | `print-product--acme` |
| `day--{norwegianDay}` | Norwegian weekday | `day--mandag` |
| `day--{englishDay}` | English weekday | `day--monday` |
| `left` \| `right` | Even/odd page number | `right` (page 1) |
| `spread` \| `single` | Whether page is linked to another | `single` |
| `has-print-page-object` | Present when placed on a print page | |
| Custom properties | From `printEdition.data.customProperties` where key starts with `.` | `.edition-{type}` → `edition-tabloid` |

Custom properties: Keys starting with `.` have the dot stripped and `{...}` placeholders replaced with the value. Example: key `.edition-{type}`, value `tabloid` → class `edition-tabloid`.

## CSS Root Properties

Set as inline `style` on the `<body>` element. Generated by `src/lib/utils.js:getCssRootProperties()`.

### Grid Properties

| Property | Description |
|----------|-------------|
| `--grid-width` | Content area width (page width minus horizontal margins) |
| `--grid-height` | Content area height (page height minus vertical margins) |
| `--grid-top` | Top margin offset |
| `--grid-left` | Left margin offset |
| `--grid-column-count` | Number of grid columns |
| `--grid-column-gap` | Gap between columns |
| `--grid-column-gaps` | Number of gaps (columns - 1) |
| `--grid-baseline` | Baseline grid interval |
| `--grid-baseline-offset` | Baseline grid offset from top |

### Page Properties

| Property | Description |
|----------|-------------|
| `--page-width` | Full page width including margins |
| `--page-height` | Full page height including margins |
| `--margin-top` | Top margin |
| `--margin-right` | Right (outside) margin |
| `--margin-bottom` | Bottom margin |
| `--margin-left` | Left (inside) margin |
| `--bleed` | Bleed width |

### Front Page Object Properties (placed editions only)

| Property | Description |
|----------|-------------|
| `--front-page-object-width` | Width of the front page object on the print page |
| `--front-page-object-height` | Height of the front page object |
| `--front-page-object-top` | Top offset of the front page object |
| `--front-page-object-left` | Left offset of the front page object |

## .frontpage Container

```scss
.frontpage {
  margin: 0 auto;
  width: var(--page-width);
  height: var(--page-height);
  padding: var(--margin-top) var(--margin-right) var(--margin-bottom) var(--margin-left);
  box-sizing: border-box;
  position: relative;
}
```

The `@page` rule in the inline `<style>` sets the PDF page size to the trim box dimensions. The `.frontpage` div fills this page.

## .content-area Container

```scss
.content-area {
  position: relative;
  height: var(--front-page-object-height, 100%);
  width: var(--front-page-object-width, 100%);
  top: var(--front-page-object-top, 0);
  left: var(--front-page-object-left, 0);
}
```

All blocks are positioned absolutely within the content area.

## Block Div

A block is a positioned container for one or more items.

```html
<div class="block block--{type} block--{modifier1} block--{modifier2} ..."
     style="--block-width: calc({width}); --block-height: calc({height}); --block-top: calc({top}); --block-left: calc({left})"
     name="{block name}">
  <!-- element divs -->
</div>
```

Source: `src/lib/frontpage.js:renderBlock()` (lines 36-49)

### Block Class Derivation

The class string is built by `classNames('block', [type, ...modifiers])`:

```
block block--{type} block--{mod1} block--{mod2} ...
```

**For group blocks** (`normalizer.js:groupToBlock()`):
- `type` = `groupData.layout` (single | row | column | box)
- `contentType` = `groupData.type`
- Modifiers from `getModifiers(groupData.modifiers, groupData.type)`:
  1. Start with `[groupData.type]` (e.g. `'article'`, `'flag'`)
  2. Append modifier entries (see [Modifier Rules](#modifier-rules))
  3. Auto-appended: `items-{count}` (number of items in group)

**For standalone items** (`normalizer.js:itemToBlock()`):
- `type` = `'single'`
- Modifiers: item modifiers + `'layer-items'` + `itemData.skin`

**For page objects** (`normalizer.js:pageObjectToBlock()`):
- `type` = `'page-object'`
- `contentType` = `object.type` (e.g. `'divider'`)
- Modifiers from `getModifiers(object.modifiers, object.type)`

### Block Style Properties

Set as CSS custom properties via `dimensions('block', block)`:

| Property | Value |
|----------|-------|
| `--block-width` | `calc({width})` |
| `--block-height` | `calc({height})` |
| `--block-top` | `calc({top})` |
| `--block-left` | `calc({left})` |

These are consumed by `frontend/styles.scss`:

```scss
.block {
  position: absolute;
  z-index: 1;
  box-sizing: border-box;
  height: var(--block-height);
  width: var(--block-width);
  top: var(--block-top);
  left: var(--block-left);
}
```

## Element (Item) Div

Each item within a block is rendered as a div:

```html
<div data-id="{item id}"
     class="item item--{mod1} item--{mod2} ..."
     style="{cssVariables}; --item-width: calc({width}); --item-height: calc({height}); --item-top: calc({top}); --item-left: calc({left}); --image-width: {imageWidth}; --image-height: {imageHeight}"
     data-movable="{movable}"
     data-resizable="{resizable}"
     data-rotatable="{rotatable}">
  <!-- inner content tree -->
</div>
```

Source: `src/lib/frontpage.js:renderBlockElements()` (lines 55-69)

### Item Class Derivation

Built by `classNames('item', modifiers)` where modifiers come from `getItemModifiers()` (`normalizer.js:305-345`):

```
item item--{mod1} item--{mod2} ...
```

Modifiers are assembled in this order (unshift = prepend, push = append):

1. Start with generic modifiers from `getModifiers(itemData.modifiers)`
2. **Prepend** `section-{slugified(section)}` if section exists
3. **Prepend** `skin-{skin}` if skin exists
4. **Prepend** `skin-{skin}-{variant}` if skinVariants[skin] exists
5. **Prepend** `skin-section-style-color-{color}` if sectionStyling.sectionColor exists
6. **Prepend** `text-pos-{textPosition}` if textPosition exists
7. **Prepend** `fixed-width` if adjustments.imageWidth exists
8. **Prepend** `fixed-height` if adjustments.imageHeight exists
9. If no image: **prepend** `text-only`
10. If image exists: **append** `image-first` (if imagePosition is `'beforeText'`) or `image-last` (otherwise)

**Example result:**
```
item item--text-pos-bottom-left item--skin-cover-story item--section-nyheter item--text-on-image item--text-color-white item--image-last
```

### Item Style Properties

| Property | Source |
|----------|--------|
| `--item-width` | `itemData.width` (freeplaced items) |
| `--item-height` | `itemData.height` |
| `--item-top` | `itemData.y` |
| `--item-left` | `itemData.x` |
| `--item-rotate` | `itemData.rotate` |
| `--image-width` | `adjustments.imageWidth` |
| `--image-height` | `adjustments.imageHeight` |
| CSS variables from modifiers | `--{key}: {value}` for modifier keys starting with `--` |
| `--title-font-size` | `adjustments.titleFontSize` (suffixed with `pt`) |

## Inner Content Tree

The HTML within each item div is generated by `normalizer.js:renderContent()`. The structure depends on `imagePosition` and `summaryPosition`.

### Default Order (imagePosition: 'afterText')

```html
<div class="header" style="--header-horizontal-offset: ...; --header-vertical-offset: ...">
  {header text}
</div>
<div class="text" style="--text-horizontal-offset: ...; --text-vertical-offset: ...">
  <div class="pretitle">{pretitle}</div>
  <div class="title" style="--font-size: {n}pt; --line-height: {n}">{title}</div>
  <div class="subheadline">{subheadline}</div>
  <div class="summary {summary--empty?}">{summary + textMeta}</div>
  <div class="byline">{byline}</div>
  {textMeta (if not inline/positional)}
</div>
<div class="image">
  <div class="image-content" style="
    background-image: url('{imageUrl}');
    --object-position-x: {focusX or adjustments};
    --object-position-y: {focusY or adjustments};
    --object-background-size: {adjustments or 'cover'};
  "></div>
  <div class="image-caption image-caption--{position} image-caption--skin-{skin}"
       style="--caption-vertical-offset: ...; --caption-horizontal-offset: ...">
    {caption text}
    <div class="image-credit">{credit}</div>
  </div>
</div>
<div class="summary summary--alt">{summary duplicate}</div>
```

### With imagePosition: 'beforeText'

The `.image` div moves before the `.text` div:

```html
<div class="header">...</div>
<div class="image">...</div>
<div class="text">...</div>
<div class="summary summary--alt">...</div>
```

### With summaryPosition: 'beforeTitle'

Content items within `.text` are reordered:

```html
<div class="text">
  <div class="summary">{summary}</div>
  <div class="byline">{byline}</div>
  {textMeta}
  <div class="pretitle">{pretitle}</div>
  <div class="title">{title}</div>
  <div class="subheadline">{subheadline}</div>
</div>
```

### Text Meta Block

The `.text-meta` block contains section and page reference:

```html
<div class="text-meta">
  <div class="section">{section}</div>
  <div class="print-section">{printSection}</div>
  <div class="page-ref">{pageRef}</div>
</div>
```

By default, text-meta appears in two places:
1. Inside the `.summary` div (at the end)
2. As a standalone element after byline in the `.text` wrapper

The `.summary .text-meta` is hidden by default CSS (`display: none`). The `.summary--alt` duplicate is also hidden (`display: none`). Theme CSS can selectively show the desired copy.

### Text Meta with refs Array

When `itemData.refs` is present, text-meta is replaced with:

```html
<div class="text-refs">
  <div class="text-ref">
    <div class="text-ref-title">{ref.title}</div>
    <div class="text-ref-page">{ref.pageRef}</div>
  </div>
  <!-- ... more refs -->
</div>
```

### textMetaPosition Variants

Controls where `.text-meta` is injected:

| Value | Behavior |
|-------|----------|
| (none / default) | After byline in `.text`; also inside `.summary` (hidden by default CSS) |
| `afterSummary` | Inside `.summary`, at the end. Not rendered as standalone after byline. |
| `summaryBefore` | Prepended before summary text inside `.summary` |
| `summaryAfter` | Appended after summary text inside `.summary` |
| `summaryInlineBefore` | Inserted at the start of the first `<p>` in summary |
| `summaryInlineAfter` | Inserted at the end of the last `<p>` in summary |
| `titleInlineBefore` | Inserted at the start of the first `<p>` in title |
| `titleInlineAfter` | Inserted at the end of the last `<p>` in title |

For inline variants, if the source contains `<p>` tags they are converted to `<div>` tags and the text-meta is injected into the first/last div.

### No-Image Rendering

When `image.url` is absent:
- The `.image` div is omitted
- A `.caption--alt` div is rendered in its place (hidden by default)
- `item--text-only` class is added to the item

### Empty Fields

Empty or falsy fields produce no HTML output. The `div()` helper returns an empty string when content is falsy. Empty summary additionally gets `summary--empty` class.

## Modifier Rules

Documented in `normalizer.js:getModifiers()` (lines 492-503):

```javascript
getModifiers(object, type)
```

1. If `type` is provided, start with `[type]`
2. Iterate `Object.entries(object)`:
   - Skip falsy values
   - Skip keys starting with `--` (reserved for CSS custom properties)
   - Boolean `true` → key name only (e.g. `text-on-image`)
   - String → `{key}-{value}` (e.g. `text-color-white`)
3. Deduplicate with `Set`

### CSS Custom Properties from Modifiers

`getCssProperties(object)` (lines 514-522) extracts `--` prefixed keys:

- Key `--gradient-start`, value `rgba(0,0,0,0.8)` → `--gradient-start: rgba(0,0,0,0.8)`

These are set as inline `style` on the item div.

## Field Classes

Generated by `getFieldClasses(object, field, cssField)` (lines 524-532):

For a field named `title`:
- Base class: `title`
- Any property in itemData matching `title--*` produces `title--{remainder}-{value}` (string) or `title--{remainder}` (boolean true)
- If content contains `<strong>`: adds `title--has-strong`

Example: `itemData['title--size'] = 'large'` → classes: `title title--size-large`

## Base CSS Layout Rules

From `frontend/styles.scss`:

| Selector | Layout |
|----------|--------|
| `.block` | `position: absolute` with CSS custom property dimensions |
| `.block--row .item` | `display: flex; flex-direction: row` with auto flex |
| `.block--column .item` | `display: flex; flex-direction: column` with auto flex |
| `.block--box .item` | `position: absolute` with CSS custom property dimensions |
| `.block--single .item` | Inherits block dimensions |
| `.item--text-only` (in row) | `flex-grow: 0` |
| `.item--fixed-width` (in row) | `flex-grow: 0` |
| `.item--fixed-height` (in column) | `flex-grow: 0; flex-shrink: 0` |
| `.image` | `flex-grow: 1; position: relative` |
| `.image-content` | `position: absolute; background-size/position via CSS vars` |
| `.summary--alt` | `display: none` (available for CSS override) |
| `.caption--alt` | `display: none` |
| `.summary .text-meta` | `display: none` |
